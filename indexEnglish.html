GomokuGo - Gomoku game robot with EV3 and LEGO

 GomokuGo

Summary
Development Process
  • Architecture


  • H/W
    • A checkerboard of 15X15 size was used. Put a rubber on the end of the syringe to hold the stalk, pierce the needle hole in the center of the rubber, and when it touches the stone, lifts up the piston and lifts the stone and the syringe into a vacuum. Then, with the EV3, the computer receives the coordinate value from the computer through TCP / IP communication, and places the pedometer on the coordinate value. I used the EV3 Python library to run the hardware.


  • Vision
    • Using the camera webcam, fix it on the checkerboard and recognize the overlay coordinates. Using the opencv library, the images are captured in real time, and the round stones that a person puts are recognized by using the Huff Circle function.
    If you recognize multiple circles at the same time as the first picture, recognition rate is lowered, so that only the newly arrived stone is recognized through the car image as shown above.


  • Algorithm
    • The weighting algorithm assigns weights according to the number of stones. For example, if two stones are horizontally, weights are weighted 2 on both sides.
    The min max algorithm is an algorithm that calculates the number of hits before using the weight used above and calculates where to put them.

    Recommendation system - Recommendation system implemented with Hadoop with Amazon product reviews

     Recommendation system

    Summary
    • When purchasing a product, users tend to view it and buy it. However, since reviews often appear in the order of the most recent review, it is difficult for people to identify which review is the right review for the item. Therefore, the project is not merely a review, but rather a program that shows more reviews of buyers.
    Development Process
    • MAP
      • 아마존 리뷰 텍스트 파일을 json파일로 불러들여서 상품번호(asin),리뷰(reviewText) 그리고 사용자들의 평가 횟루를 나타내는 helpful의 분모를 추출한다. helpful의 분자는 긍정적인 리뷰평을 한 구매자의 수를 나타내고, 분모는 리뷰평에 참여한 구매자의 수를 나타낸다. Map의 context.write에서 key값은 “asin”을 사용하고 value값은 “helpful”의 분모와 “reviewText”를 세미콜론(;)을 경계로 합친 Text형식으로 Reduce로 보낸다. 
    • REDUCE
      • Map으로부터 key(“asin”)와 value(“helpful” + “reviewText”)를 받는다. 이때 value를 “helpful”과 “reviewText”를 세미콜론(;)을 경계로 split을 사용하여 구분한다. Text형식인 “helpful”을 Integer형으로 변환하여 같은 key(“asin”)중에 최대값을 추출하고 그 key(“asin”)의 대한 “reviewText”를 출력한다.
